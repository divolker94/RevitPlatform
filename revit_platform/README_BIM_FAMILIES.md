# BIM Семейства - Инструкция по настройке

## Описание
Система для отображения BIM семейств Revit с изображениями и техническими характеристиками, загруженными из PostgreSQL базы данных.

## Структура проекта

### Frontend (React)
- `src/components/BimFamilyCard.js` - Карточка семейства
- `src/components/BimFamilyCard.css` - Стили карточки
- `src/pages/Families.js` - Страница списка семейств
- `src/pages/Families.css` - Стили страницы списка
- `src/pages/BimFamilyDetail.js` - Страница детального просмотра
- `src/pages/BimFamilyDetail.css` - Стили детального просмотра

### Backend (Django)
- `backend/bim_families/` - Django приложение
  - `models.py` - Модели данных
  - `serializers.py` - API сериализаторы
  - `views.py` - API представления
  - `urls.py` - URL маршруты
  - `admin.py` - Админ панель

## Установка и настройка

### 1. Backend (Django)

#### Создание миграций:
```bash
cd revit_platform/backend
python manage.py makemigrations bim_families
python manage.py migrate bim_families
```

#### Регистрация в settings.py:
Убедитесь, что `bim_families` добавлено в `INSTALLED_APPS` в `config/settings.py`.

#### Регистрация в urls.py:
Убедитесь, что URL для BIM семейств добавлен в `config/urls.py`.

### 2. Frontend (React)

#### Установка зависимостей:
```bash
cd revit_platform/frontend
npm install
```

#### Запуск:
```bash
npm start
```

## Использование

### API Endpoints

#### Получение списка семейств:
```
GET /api/bim-families/
```

#### Получение конкретного семейства:
```
GET /api/bim-families/{id}/
```

#### Получение изображений семейства:
```
GET /api/bim-families/{id}/images/
```

#### Поиск и фильтрация:
```
GET /api/bim-families/?search=окно&category=двери
```

### Фронтенд маршруты

- `/families` - Список всех BIM семейств
- `/bim-families/{id}` - Детальный просмотр конкретного семейства

## Структура данных

### BimFamily
- `name` - Название семейства
- `external_id` - Внешний ID
- `url` - Ссылка на источник
- `description` - Описание
- `technical_specs` - Технические характеристики (JSONB)
- `basic_specs` - Основные характеристики (JSONB)
- `catalog_items` - Каталог элементов (массив)
- `company_info` - Информация о компании (JSONB)
- `download_info` - Информация для скачивания (JSONB)
- `parsing_method` - Метод парсинга
- `total_images` - Количество изображений
- `parsed_at` - Дата парсинга

### FamilyImage
- `family` - Ссылка на семейство
- `local_path` - Путь к изображению
- `alt_text` - Альтернативный текст
- `title` - Заголовок
- `image_type` - Тип изображения
- `width`, `height` - Размеры
- `file_size` - Размер файла
- `local_filename` - Имя файла

## Изображения

### Структура папок
Изображения должны находиться в подпапках по названиям семейств:
```
frontend/public/images/bim_families/
├── 18333_Каталог фундаментов и подпорных сооружений. Revit/
│   ├── image1.jpg
│   ├── image2.png
│   └── ...
├── Другое семейство/
│   ├── image1.jpg
│   └── ...
└── ...
```

### Формат путей в базе
В базе данных пути могут храниться как:
```
images/bim_families/18333_Каталог фундаментов и подпорных сооружений. Revit/image1.jpg
```
или просто как имя файла:
```
image1.jpg
```

Система автоматически определит правильный путь на основе структуры папок.

### Обработка ошибок
При ошибке загрузки изображения отображается placeholder с иконкой.

## Возможные проблемы

### 1. Изображения не загружаются
- Проверьте пути к изображениям
- Убедитесь, что папка `images/bim_families/` существует
- Проверьте права доступа к файлам

### 2. API не отвечает
- Проверьте, что Django сервер запущен
- Убедитесь, что приложение `bim_families` добавлено в `INSTALLED_APPS`
- Проверьте URL маршруты

### 3. Ошибки миграций
- Удалите папку `migrations/` и создайте заново
- Проверьте совместимость моделей с существующей базой
